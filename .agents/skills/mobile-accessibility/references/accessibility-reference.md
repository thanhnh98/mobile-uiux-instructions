# Accessibility Reference

## Targets
- Interactive targets meet platform minimum size.
- Adjacent targets have enough spacing.
- Custom controls behave like native controls.

## Focus Order
- Focus follows task order.
- Related content is grouped when useful.
- Modals and sheets trap or restore focus appropriately.

## Labels
- Icon-only controls have accessible names.
- Inputs include labels, errors, and hints.
- Dynamic values expose current state.
- Decorative images are hidden from assistive tech.

## Contrast
- Text and essential icons meet contrast requirements.
- Disabled states remain understandable.
- Status is not communicated by color alone.

## Text Scaling
- Large text does not truncate critical information.
- Buttons, tabs, and cards survive long strings.
- Layout can scroll instead of clipping.

## Screen Reader
- Loading, errors, completion, and validation changes are announced.
- Lists expose item count or position when helpful.
- Controls use clear role, value, and state semantics.

## Motion
- Reduced motion is respected.
- Animations are not required to understand state.
- No flashing or rapid motion patterns.

## Gesture Alternatives
- Swipe, drag, long-press, and pinch actions have alternatives.
- Destructive gestures require confirmation or undo where appropriate.
